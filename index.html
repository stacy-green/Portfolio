One-Stop One-Shot

OSOS is my capstone project for my Python-Based Fullstack Developer Bootcamp. The project contains 4 separate apps: OSOS, Encounter Builder, Map Generator, and Villain/Antagonist generator.

OSOS builds a portfolio for a single session of Dungeons and Dragons 5th Edition. It allows Dungeon Masters to quickly build a standalone scenario when scheduling conflicts prevent a player from attending, but everyone else already showed up.

Encounter Builder

The Encounter Builder app leans heavily on Vuejs. Character levels are stored locally, and experience goals are calculated as party levels change. Users choose monsters for encounters by first selecting the Challenge Rating from the dropdown menu, then selecting a monster from a list that is populated on CR selection. Finally, the selected monster's statblock is populated on the right, using Vuejs to fill in each item of information in an HTML template. Users can scroll up or down on the statblock depending on how much information is present.

Map Generator

Map Generator is a bit of a misnomer. It actually randomly slects a map after selecting a size; each map size has 15 maps to draw from the database. Users can add as many maps as they want, for a longer or shorter play session.

Villain Generator

The Villain/Antagonist Generator is another app that leans heavily on Vuejs. The only required fields are first name, gender, and race. All official races and names are loaded into a json file. Users can enter their own data for their villain, or randomly select by clicking "roll." The villain generator includes fields such as organization, motivation, and secret, so that Dungeon Masters can create and customize robust villains almosts instantly.

One-Shot Portfolio

The OSOS app, the project's namesake, is the portfolio itself. The portfolio is a summary of the session, giving the Dungeon Master a quick snapshot of the maps, antagonists, and combat encounters. However, clicking on practically any element takes the user to more detailed versions. The buttons are self-explanatory; users can click on each button and find themselves on the respective app.
